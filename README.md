## 🇹🇷 Türkçe

### Proje Hakkında
Bu proje, BİL 2004 Veritabanı Yönetimi dersi kapsamında, bir kafenin günlük operasyonlarını (sipariş yönetimi, stok takibi, personel ve müşteri kayıtları) dijital ortama aktarmak amacıyla SQL Server (SSMS) üzerinde geliştirilmiş ilişkisel bir veritabanı yönetim sistemidir (RDBMS). 

**Hazırlayan:** Özhan YANA

> ⚠️ **Veri Sorumluluk Reddi (Disclaimer):** > Bu projede yer alan `Veriler.sql` içerisindeki tüm veriler (müşteri isimleri, iletişim bilgileri, personel kayıtları, sipariş detayları vb.) sistemin çalışabilirliğini test etmek amacıyla **tamamen kurgusal olarak tarafımızca oluşturulmuştur**. Gerçek kişi, kurum veya ticari verileri yansıtmamaktadır.

### Veritabanı Mimarisi ve Özellikler
- **İlişkisel Tasarım:** Müşteriler, Personel, Kategoriler, Ürünler, Siparişler, Sipariş Detayları ve Ödemeler olmak üzere 7 farklı tablo, Foreign Key ve Primary Key kısıtlamalarıyla mimariye uygun şekilde birbirine bağlanmıştır.
- **Veri Bütünlüğü (Data Integrity):** Stok miktarının sıfırın altına düşmemesi veya fiyatların negatif olmaması gibi iş kuralları `CHECK` kısıtlamaları ile güvence altına alınmıştır.
- **Gelişmiş Sorgular:** Veritabanı üzerinde `JOIN`, `GROUP BY`, `HAVING`, alt sorgular (subqueries) ve tarih fonksiyonları kullanılarak çeşitli iş zekası (BI) analizleri yapılmıştır (Örn: En çok sipariş alan personel, stok maliyeti analizi, saatlere göre sipariş yoğunluğu vb.).

## 🇬🇧 English

### About the Project
This project is a relational database management system (RDBMS) developed on SQL Server (SSMS) for the BİL 2004 Database Management course. It is designed to digitize the daily operations of a cafe, including order management, inventory tracking, and staff/customer records.

**Author:** Özhan YANA

> ⚠️ **Data Disclaimer:** > All data populated in the `Veriler.sql` file (such as customer names, contact info, staff records, order details, etc.) is **mock data uniquely generated by us** solely for testing the database functionality. It does not reflect any real-world individuals, organizations, or commercial figures.

### Database Architecture and Features
- **Relational Design:** A total of 7 tables (Customers, Staff, Categories, Products, Orders, Order Details, and Payments) are interconnected using Foreign Key and Primary Key constraints.
- **Data Integrity:** Business rules, such as preventing negative stock levels or prices, are enforced using `CHECK` constraints.
- **Advanced Queries:** Various business intelligence (BI) analyses were performed on the database using `JOIN`s, `GROUP BY`, `HAVING`, subqueries, and date functions (e.g., top-performing staff, stock cost analysis, hourly order density).

### Repository Structure
- [**Tablolar.sql**](./Tablolar.sql): The DDL script containing the database schema and table relationships.
- [**Veriler.sql**](./Veriler.sql): The DML script used to insert mock/synthetic data into the tables.
- [**Sorgular.sql**](./Sorgular.sql): Analytical and operational SQL queries demonstrating the cafe's workflow.
- [**BİL 2004 Veritabanı Yönetimi Proje Raporu.pdf**](./BİL%202004%20Veritabanı%20Yönetimi%20Proje%20Raporu.pdf): The project report containing the ER diagram, requirements analysis, and project details.
